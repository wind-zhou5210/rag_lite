# RAG Lite 项目开发宪法（constitution.md）

Version: 1.0
Ratified: 2026-03-02

本文件定义了本项目不可动摇的核心开发原则。所有 AI Agent 在进行技术规划和代码实现时，必须无条件遵循。

---

## 第一条：简单性原则 (Simplicity First)

**核心：** 在满足业务需求的前提下，追求**最小复杂度**与**最少认知负担**，避免任何非必要的设计、抽象或依赖。

- **1.1 (YAGNI):** 只实现 `spec.md` 中明确要求的功能。
- **1.2 (框架内优先):** 必须优先使用 Python 标准库和 Flask 已有机制，其次才是第三方库。
- **1.3 (反过度工程):** 简单的函数和数据结构优于复杂的接口和继承体系。

---

## 第二条：测试先行铁律 (Test-First Imperative) - 不可协商

**核心：** 所有新功能或 Bug 修复，都必须从编写一个（或多个）失败的测试开始。

- **2.1 (TDD 循环):** 严格遵循 "Red-Green-Refactor" 循环。
- **2.2 (参数化测试):** 使用 pytest 的 `@pytest.mark.parametrize` 进行参数化测试，覆盖多种输入场景。
- **2.3 (真实依赖):** 优先编写集成测试，使用真实的数据库和依赖，而非过度 mock。

---

## 第三条：明确性原则 (Clarity and Explicitness)

**核心：** 代码的首要目的是让人类易于理解。

- **3.1 (错误处理):** **不可协商**：禁止吞异常，禁止 `except: pass`，所有异常必须明确业务语义或被统一异常机制转换为可审计的错误响应。
- **3.2 (无全局变量):** 绝不允许使用全局变量来传递状态，所有依赖必须通过函数参数或类的属性传递。
- **3.3 (类型提示):** 公共函数和类方法必须使用 Python 类型提示（Type Hints）。

---

## 第四条：项目规范遵循 (Project Conventions)

**核心：** 严格遵循项目既有的架构模式和代码规范。

- **4.1 (服务层模式):** 业务逻辑封装在 `services/` 层，路由只负责请求解析和响应组装。
- **4.2 (返回值规范):** 服务层方法返回 `(result, error_message)` 元组，成功时 `error_message` 为 `None`。
- **4.3 (事务管理):** 使用 `session_scope()` 上下文管理器处理数据库事务。
- **4.4 (统一响应):** API 响应使用 `app/util/response.py` 中的 `success()`, `error()` 等函数。

---

## 治理 (Governance)

本宪法具有最高优先级，其效力高于任何 `CLAUDE.md` 或单次会话中的指令。
