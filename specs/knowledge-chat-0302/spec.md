# 知识库多轮对话问答 需求规范

Version: 1.0
Created: 2026-03-02
Status: Draft

## 1. 概述

### 1.1 背景
RAG Lite 系统目前已支持知识库管理和文档上传功能，但缺少核心的知识问答能力。用户上传文档后，无法与知识库进行交互式对话。需要增加知识库聊天功能，让用户能够基于自己的知识库进行智能问答。

### 1.2 目标
- 提供基于 RAG 技术的智能问答能力
- 支持多轮对话，记住上下文，允许用户追问
- 展示回答来源，增强回答的可信度和可追溯性
- 提供良好的对话管理体验

### 1.3 范围

**MVP 范围**：
- 知识库选择器
- 多轮对话（保留 10 轮上下文）
- 流式输出回答
- 来源展示（文档名称 + 片段）
- 对话管理（创建、切换、删除）
- 对话历史永久保存

**后续迭代**：
- 对话分享功能
- 对话导出功能
- 回答点赞/点踩反馈
- 多知识库联合问答

### 1.4 成功指标
暂不设定量化指标，以功能完整性为主要目标。

## 2. 用户分析

### 2.1 目标用户
- 已注册并登录的系统用户
- 已创建知识库并上传文档的用户
- 需要从知识库中快速获取信息的用户

### 2.2 用户故事

**作为** 知识库所有者
**我想要** 与我的知识库进行对话问答
**以便于** 快速获取文档中的信息，而无需逐个阅读文档

**作为** 知识库所有者
**我想要** 看到回答的来源
**以便于** 验证回答的准确性并追溯原始文档

**作为** 知识库所有者
**我想要** 管理我的对话历史
**以便于** 回顾之前的问答或清理不需要的对话

### 2.3 用户旅程

```
1. 用户进入聊天页面
2. 从下拉列表选择一个知识库
3. 输入问题并发送
4. 系统流式输出回答，并展示引用来源
5. 用户继续追问（系统记住上下文）
6. 用户可随时切换到其他对话或创建新对话
7. 用户可删除不需要的对话
```

## 3. 功能需求

### 3.1 核心功能

| 功能点 | 描述 | 优先级 | 验收标准 |
|--------|------|--------|----------|
| 知识库选择 | 下拉列表选择知识库 | P0 | 只显示当前用户创建的知识库 |
| 多轮对话 | 支持上下文记忆 | P0 | 保留最近 10 轮对话历史 |
| 流式输出 | 打字机效果展示回答 | P0 | 逐字/逐句输出，体验流畅 |
| 来源展示 | 显示引用的文档和片段 | P0 | 折叠展示，可展开查看详情 |
| 对话列表 | 左侧显示对话历史 | P0 | 显示标题、时间，可切换/删除 |
| 对话管理 | 创建/切换/删除对话 | P0 | 新建对话自动生成标题 |
| 历史保存 | 永久保存对话记录 | P0 | 刷新页面后可继续 |

### 3.2 功能详细说明

#### 功能 A：知识库选择器
- **触发条件**：用户进入聊天页面时
- **操作流程**：
  1. 页面加载时获取用户的知识库列表
  2. 显示在下拉选择器中
  3. 用户选择后，加载该知识库的对话列表
- **业务规则**：
  - 只显示用户自己创建的知识库
  - 显示知识库名称和文档数量
- **输出结果**：选定知识库后，可开始对话

#### 功能 B：多轮对话
- **触发条件**：用户发送问题
- **操作流程**：
  1. 用户在输入框输入问题
  2. 点击发送或按 Enter
  3. 系统进行 RAG 检索 + LLM 生成
  4. 流式输出回答
  5. 显示引用来源
- **业务规则**：
  - 保留最近 10 轮对话作为上下文
  - 超过 10 轮时，移除最早的对话
  - 用户问题和 AI 回答各算一轮
- **输出结果**：带来源的回答

#### 功能 C：来源展示
- **触发条件**：AI 回答包含引用来源
- **展示内容**：
  - 文档名称
  - 引用的文本片段
- **展示形式**：
  - 默认折叠状态
  - 点击"查看来源"展开
  - 可同时展示多个来源

#### 功能 D：对话列表
- **触发条件**：选择知识库后自动加载
- **展示内容**：
  - 对话标题（自动生成或用户自定义）
  - 创建/更新时间
- **操作**：
  - 点击切换到该对话
  - 删除按钮（带确认）
  - 新建对话按钮

#### 功能 E：对话管理
- **新建对话**：
  - 点击"新建对话"按钮
  - 自动生成标题（根据第一个问题截取，最多 20 字）
  - 用户可编辑标题
- **切换对话**：
  - 点击列表中的对话即可切换
  - 自动加载该对话的历史消息
- **删除对话**：
  - 点击删除按钮
  - 弹出确认提示
  - 确认后删除对话及所有消息

### 3.3 交互说明

**页面布局**：
```
+------------------+------------------------+
|                  |                        |
|  知识库选择器     |     聊天消息区域        |
|                  |                        |
+------------------+                        |
|                  |                        |
|  对话列表         |                        |
|  - 对话1          |                        |
|  - 对话2          |                        |
|  - ...           |                        |
|                  |                        |
|  [新建对话]       |                        |
|                  +------------------------+
|                  |  输入框    [发送]       |
+------------------+------------------------+
```

**状态说明**：
- 空状态：选择知识库后无对话时，显示引导提示
- 加载中：发送问题后显示加载动画
- 流式输出：逐字显示回答内容
- 错误状态：显示友好的错误提示

## 4. 数据需求

### 4.1 数据展示

**知识库列表**：
| 字段 | 说明 |
|------|------|
| id | 知识库 ID |
| name | 知识库名称 |
| document_count | 文档数量 |

**对话列表**：
| 字段 | 说明 |
|------|------|
| id | 对话 ID |
| title | 对话标题 |
| created_at | 创建时间 |
| updated_at | 最后更新时间 |

**聊天消息**：
| 字段 | 说明 |
|------|------|
| id | 消息 ID |
| role | 角色（user/assistant） |
| content | 消息内容 |
| sources | 来源列表（仅 assistant） |
| created_at | 创建时间 |

**来源信息**：
| 字段 | 说明 |
|------|------|
| document_name | 文档名称 |
| chunk_content | 引用片段 |
| score | 相似度得分（可选展示） |

### 4.2 数据操作

| 操作 | 说明 |
|------|------|
| 创建对话 | POST /api/chat/conversations |
| 获取对话列表 | GET /api/chat/conversations?kb_id=xxx |
| 获取对话详情 | GET /api/chat/conversations/{id} |
| 更新对话标题 | PUT /api/chat/conversations/{id} |
| 删除对话 | DELETE /api/chat/conversations/{id} |
| 发送消息 | POST /api/chat/query（流式） |

### 4.3 数据校验规则

| 字段 | 规则 |
|------|------|
| 问题输入 | 1-2000 字符，不能为空 |
| 对话标题 | 1-50 字符，可选（自动生成） |

## 5. 非功能需求

### 5.1 性能要求
- 首字输出时间（TTFT）：< 3 秒
- 流式输出流畅，无明显卡顿
- 对话列表加载：< 1 秒

### 5.2 兼容性要求
- 浏览器：Chrome、Firefox、Edge、Safari 最新两个大版本
- 屏幕分辨率：最小支持 1024x768

### 5.3 安全要求
- 用户只能访问自己创建的知识库和对话
- 对话内容通过 JWT 认证保护
- 敏感数据传输使用 HTTPS

## 6. 边缘情况与异常处理

| 场景 | 处理方式 |
|------|----------|
| 知识库无文档 | 显示提示："该知识库暂无文档，请先上传文档后再进行问答" |
| 知识库处理中 | 可正常使用，基于已处理完成的文档进行问答 |
| 未找到相关内容 | 回答："抱歉，在知识库中未找到与您问题相关的内容，请尝试换一种方式提问" |
| 对话为空 | 显示引导提示："开始新对话，输入您的问题" |
| 网络异常 | 显示错误提示，支持重试 |
| LLM 服务异常 | 显示友好错误提示："服务暂时不可用，请稍后重试" |
| 删除确认 | 弹出确认框："确定删除该对话？删除后无法恢复" |

## 7. 验收标准

### 7.1 功能验收
- [ ] 可选择知识库并开始对话
- [ ] 多轮对话能正确记住上下文
- [ ] 回答以流式方式输出
- [ ] 来源信息正确展示并可折叠
- [ ] 对话列表正确显示
- [ ] 可创建、切换、删除对话
- [ ] 刷新页面后对话历史保留
- [ ] 边缘情况处理正确（无文档、未找到等）

### 7.2 体验验收
- [ ] 交互流畅，无明显卡顿
- [ ] 错误提示友好清晰
- [ ] 流式输出效果自然
- [ ] 符合用户操作习惯

### 7.3 安全验收
- [ ] 无法访问其他用户的知识库
- [ ] 无法访问其他用户的对话

## 8. 附录

### 8.1 API 接口设计

#### 创建对话
```
POST /api/chat/conversations
Request: { "kb_id": "知识库ID", "title": "标题（可选）" }
Response: { "code": 200, "data": { "id": "对话ID", "title": "...", ... } }
```

#### 获取对话列表
```
GET /api/chat/conversations?kb_id=xxx
Response: { "code": 200, "data": [{ "id": "...", "title": "...", "created_at": "..." }] }
```

#### 发送消息（流式）
```
POST /api/chat/query
Request: { "conversation_id": "对话ID", "question": "用户问题" }
Response: Server-Sent Events (SSE)
  data: {"type": "content", "content": "回答内容..."}
  data: {"type": "source", "source": {...}}
  data: {"type": "done"}
```

#### 更新对话标题
```
PUT /api/chat/conversations/{id}
Request: { "title": "新标题" }
Response: { "code": 200, "message": "success" }
```

#### 删除对话
```
DELETE /api/chat/conversations/{id}
Response: { "code": 200, "message": "success" }
```

### 8.2 数据库设计

**对话表 (conversation)**：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键 UUID |
| kb_id | VARCHAR(36) | 知识库 ID |
| user_id | VARCHAR(36) | 用户 ID |
| title | VARCHAR(100) | 对话标题 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**消息表 (message)**：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键 UUID |
| conversation_id | VARCHAR(36) | 对话 ID |
| role | ENUM | user/assistant |
| content | TEXT | 消息内容 |
| sources | JSON | 来源信息（JSON 数组） |
| created_at | DATETIME | 创建时间 |
